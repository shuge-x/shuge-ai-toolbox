## 整体评估

**拆分方向建议**：

将任务拆分为两个主要组：

1. **摘要算法组**（优先实现）：
   - 停用词列表实现
   - 句子提取函数实现
   - 摘要长度控制
   - 边界条件处理（空输入、空结果）

2. **UI 组件组**（依赖算法组）：
   - TextSummaryPage 组件基本结构
   - 输入区渲染
   - 控制区渲染（长度选择、按钮）
   - 结果展示区
   - 空输入/空结果提示

**优先级排序**：

1. `extractSummary` 函数（核心算法，优先 TDD）
2. 停用词列表（算法依赖）
3. TextSummaryPage 组件（UI 层，依赖算法）

**粒度标准**：

- 每个 task 包含一个测试文件 + 一个实现文件
- 工具函数 task：extractSummary、stopwords 各自独立
- UI 组件 task：组件整体作为一个 task（内部状态简单）
- 最小实现：每步只实现能通过测试的最少代码
